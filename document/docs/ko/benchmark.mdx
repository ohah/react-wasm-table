# 벤치마크

import { Benchmark } from "demo";

동일한 데이터·동일한 컬럼으로 **나란히 벤치마크**할 수 있는 페이지를 제공합니다.

## 비교 대상

|             | react-wasm-table                    | @tanstack/react-table         |
| ----------- | ----------------------------------- | ----------------------------- |
| 렌더링      | Canvas (WASM)                       | DOM + @tanstack/react-virtual |
| 행 처리     | 전체 데이터 메모리, 코어에서 가상화 | DOM에서 가상 스크롤           |
| 동일 데이터 | ✓ `generateEmployees(N)`            | ✓ 동일                        |

## 측정 항목

- **초기 렌더 (ms)** — 데이터 설정 후 첫 페인트까지 소요 시간(낮을수록 좋음).
- **스크롤 FPS** — 스크롤 시 데모 페이지 우측 상단 FPS로 두 그리드의 스크롤 부드러움을 비교할 수 있습니다.

## 벤치마크 실행

행 수(10K~1M)를 선택하고 **벤치마크 실행**을 누르면 같은 데이터가 두 테이블에 로드되며, 측정된 초기 렌더 시간이 표시됩니다.

## 빠른 이유

- **Canvas** — 셀당 DOM 노드 없음; 하나의 캔버스가 보이는 영역만 그림.
- **Rust/WASM** — 레이아웃(Taffy), 히트 테스트, 스크롤 연산이 WebAssembly에서 실행.
- **가상화** — 보이는 행만 레이아웃·그리기하고, 나머지는 JS 메모리에만 유지.
